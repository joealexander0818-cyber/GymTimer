# GymTimer 需求文档

## 1. 平台
- **Web App (PWA)**：iPhone 浏览器使用，支持添加到主屏幕
- **提醒方式**：铃声提醒（iOS Safari 震动 API 支持有限）

## 2. 核心功能

### 2.1 训练模板预设
- 预设3个训练日模板：推力日、拉力日、腿部训练日
- 每个模板包含多个动作，每个动作配置：
  - 动作名称
  - 组数（默认4组）
  - 休息时间（大肌群90秒，小肌群60秒）
- 支持编辑模板、新增自定义模板

### 2.2 训练流程

**开始训练：**
1. 选择训练日模板
2. 显示第一个动作名称、上次训练重量（如有）

**训练中：**
1. 点"完成本组" → 开始倒计时休息
2. 倒计时结束前10秒震动（准备提醒）
3. 倒计时结束震动（正式提醒）→ 自动进入下一组
4. 重复直到本动作全部组数完成

**动作切换：**
- 本动作全部组数完成后，自动提示"下一个动作：xxx，是否开始？"
- 用户确认后进入下一个动作

**结束训练：**
- 全部动作完成 → 提示确认结束
- 用户确认后保存记录（避免用户临时想加动作时误结束）

### 2.3 重量记录
- 每组动作可记录重量（kg）
- 默认填入上次该动作的重量，用户可手动修改
- 支持每组记录不同重量（如递增/递减）

### 2.4 训练后笔记
- 文字笔记，选填
- 用于记录感悟、问题、下次调整等
- 不强制填写，可直接结束

### 2.5 历史记录
- 按日期查看每次训练详情
- 查看每个动作的历史重量变化
- 支持查看训练后笔记

## 3. 用户体验要点
- 操作简洁，减少训练中断
- 震动提醒清晰，不看屏幕也能感知
- 历史记录直观展示进步轨迹

## 4. 数据结构（初步）

**训练模板：**
```
Template
├── name: String (推力日/拉力日/腿部训练日)
├── exercises: [Exercise]
    ├── name: String
    ├── sets: Int (默认4)
    └── restSeconds: Int (60 or 90)
```

**训练记录：**
```
WorkoutSession
├── date: Date
├── templateName: String
├── exercises: [ExerciseRecord]
    ├── name: String
    ├── weight: [Double] (每组的重量)
    └── completedSets: Int
└── note: String? (训练后笔记)
```
